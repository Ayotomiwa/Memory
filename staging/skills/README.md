# staging/skills/

Proposed new skills before they are added to `.claude/skills/`.

Naming: `YYYY-MM-DD-<skill-name>.md`

## Process

1. Drop a proposal here using `_template.md`.
2. The weekly lint pass checks each proposal against existing `.claude/skills/` entries and flags duplicates, overlaps, or contradictions.
3. The lead reviews the lint output and decides: approve, reject, or merge with an existing skill.
4. If approved, the lead creates `.claude/skills/<name>/SKILL.md` and adds an entry to `claude-assets/skills-catalog.md`.
5. The staging file stays as audit trail.

Skills are human-authored. Claude surfaces conflicts; humans decide.
